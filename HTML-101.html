HTML 101 - a primer

This HTML page contains some key pointers for making HTML pages.

Elements and tags

HTML is a Markup Language that uses tags to "mark up" your content.

An element is the tag and it's content, like this:

        <h1>My title</h1>

Here <h1> is the tag (with </h1> as the closing tag), and "My title" is the content.

Attribute

An attribute gives more info or function to the elements (tags), like this:

        <h1 lang="en">My title</h1>

Here the "lang" is an attribute, telling the browser that the content of this element is in English.

Nested tags

Nested tags are when you use a tag within a tag, like this, where the <h1> is nested within the <body>: 

        <body>
            <h1>My title</h1>
        </body>

Whenever you do this make sure that child tags are closed before parents. If not, and you try to close the body tag before the h1, you will get in all sorts of problems; so don't do this:

        <body>
            <h1>My title
        </body>
        </h1>

Headings

Headings are important in HTML documents. There should only be one <h1> tag on each page; or, more spesifically, only one per equal important section on the page, for multi-section pages.

Then use <h2> for the next level, <h3> for the next, and so on (until <h6> if needed). This establishes a semantic structure (content hierarchy) for the content on your page.

Use Headers for structure, not formatting

Don't use headings to make text BIG or bold. Use style/stylesheets to format the headings, if needed, as you do with all other content on your page (ie. if you want the h3 to be larger, do not substitute it with a h2 just because that is bigger by default).

Paragraphs

Use <p> for making paragraph tags on your page. This is the standard body text ("brødtekst" in Norwegian) for content on your page, one tag per paragraph ("avsnitt"), and problably - along with <div> perhaps - one of the most often used tags in HTML.

Note that even though the closing paragraph tag technically are optional, it's best practice to always include the closing tag, </p>.

Line breaks

Sometimes you don't want a whole new paragraph, but you ned a new line. To do this, you add a line break, with the <br> tag, which do not need a closing tag, like this:
This is on a separate line,
and this on another new line,
and another one, but all are part of the same paragraph.

Note that in the xhtml days you had to close all tags, and then some tags were re-defined as selfclosing, and you might find examples of this, with line break tag (and others) looking like this: <br/> or <br />.

Emphazising tekst

There are two different ways of emphasizing text, by making it bolder or italic, and HTML contains two ways of doing this, the old, typographical way, and the more modern, semantic way. You can use <b> and <i> to make text bolder or italic; or you can use <strong> and <em> to ephazise the meaning. The strong tag will display as bold, while the em tag will use italics, in most browsers; but for instance web readers (uses by disabled and others) will differentiate these.

Images

Images are placed onto an HTML site using the <img> tag. The image tag is of a kind called replaced content, which means that whenever a browser encounters it, it will stop and replace the tag with whatever it references. To make the reference the image tag has one all important attribute, src, to point to the source file. The other mandatory attribute for images is the alt tag that should be used to describe what's on the photo. There's also a few optional attributes, like target, width and height.

The image tag should look like this <img src="path/to/image.kind" alt="Desciption og what's shown on the image"> (note that this is a non closing tag, so you do not need a </img>).

The src attribute points to the image, and usually it will do this relative to the HTML file that includes it, so: If the images are in a folder called "media" on the same level at the HTML file the path is "media/your-image.kind". On more complex setups the path can be longer.

Also, if you set both width and height, make sure you retain the same aspect ratio to protect the image propotions.

Common image types

Technically not a part of HTML, it's still useful to know the most common image types used on the web:

JPEG

JPEG (pron. JAY-PEG) is a commonly used method of lossy compression for digital images, particularly for those images produced by digital photography. The degree of compression can be adjusted, allowing a selectable tradeoff between storage size and image quality.

GIF

The Graphics Interchange Format, better known by its acronym GIF (pron. JIF or GIF [1][2][3]) is a bitmap image format that supports up to 8 bits per pixel for each image, allowing a single image to reference its own palette of up to 256 different colors chosen from the 24-bit RGB color space. It also supports animations and allows a separate palette of up to 256 colors for each frame. These palette limitations make GIF less suitable for reproducing color photographs and other images with color gradients, but it is well-suited for simpler images such as graphics or logos with solid areas of color.

PNG

Portable Network Graphics, og PNG (pron. PING) is a raster graphics file format that supports lossless data compression. PNG was created as an improved, non-patented replacement for Graphics Interchange Format (GIF), and is the most widely used lossless image compression format on the Internet. PNG supports palette-based images (with palettes of 24-bit RGB or 32-bit RGBA colors), grayscale images (with or without alpha channel for transparency), and full-color non-palette-based RGB/RGBA images (with or without alpha channel).

SVG

Scalable Vector Graphics (SVG) is an XML-based vector image format for two-dimensional graphics with support for interactivity and animation. The SVG specification is an open standard developed by the World Wide Web Consortium (W3C) since 1999.

SVG images and their behaviors are defined in XML text files. This means that they can be searched, indexed, scripted, and compressed. As XML files, SVG images can be created and edited with any text editor, as well as with drawing software.

Forms

The <form> tag is used to create an HTML form for user input. The <form> element can contain one or more of the following form elements:

  • <input> specifies an input field where the user can enter data, the default is text, but it there's a wide range of input type attributes.
  • <textarea> defines a multi-line text input control, that can hold an unlimited number of characters
  • <button> defines a clickable button, inside of which you can put content, like text or images (giving you more options than the input button type)
  • <select> element is used to create a drop-down list
  • <option> defines an option in a select list.
  • <optgroup> is used to group related options in a drop-down list. If you have a long list of options, groups of related options are easier to handle for a user
  • <fieldset> is used to group related elements in a form, by drawing a box around the related elements
  • <label> defines a label for an input element, and using the for attribute you may reference the input element, making the label clickable

HTML Block and Inline Elements

Block-level Elements

A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can).

This <div> element is a block-level element.

Examples of block-level elements:

  • <div>
  • <h1> - <h6>
  • <p>
  • <form>

Inline Elements

An inline element does not start on a new line and only takes up as much width as necessary. This is an inline <span> element inside a paragraph. Examples of inline elements:

  • <span>
  • <a>
  • <img>

Special Characters

Special characters are charachters like [] < > {} " " ' ' & ™ © and might also include others, like scandinavian characters, æ, ø and å, like in "blåbærsyltetøy". We've already seen examples of this on the page, with all the tags using the < and > special characters. To display these consistantly, we use HTML entities for some (if not all) of these. Some of them are strongly recommended, and should be used according to the standard, like <, > and & because they can break your page, others aren't necessary, but to should be used to make sure the correct character are displayed across different browsers and languages.

Some of the most used ones are:

  • & : &amp;
  • < : &lt;
  • > : &gt;
  • @ : &commat;
  • ™ : &trade;
  • © : &copy;
  • ® : &reg;
  • # : &num;
  • æ / Æ : &aelig; / &AElig;
  • ø / Ø : &oslash; / &Oslash;
  • å / Å : &aring; / &Aring;

Both <q> and <blockquote> are used for quotations. The <q> tag defines a short quotation, while the <blockquote> tag specifies a section that is quoted from another source.

Structuring elements

HTML let us stucture elements based on their meaning, also referred to as semantic. For this we use the sectioning Elements, <h1>-<h6>, <article>, <aside>, <nav>, and <section>. We also have the semantic elements, <header> (not to be confused with <head>), <main>, and <footer>.

Nav

The <nav> tag defines a set of navigation links. Notice that NOT all links of a document should be inside a <nav> element. The <nav> element is intended only for major block of navigation links.

Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

Article

The <article> tag specifies independent, self-contained content. An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

Section

The <section> tag defines sections in a document, such as chapters, headers, footers, or any other sections of the document.

Aside

The <aside> tag defines some content aside from the content it is placed in. The aside content should be related to the surrounding content.

Footer

The <footer> tag defines a footer for a document or section. A <footer> element should contain information about its containing element.

A <footer> element typically contains:

  • authorship information
  • copyright information
  • contact information
  • sitemap
  • back to top links
  • related documents

You can have several <footer> elements in one document.

Div and Span

The <div> tag defines a generic division or a section in an HTML document. The <div> tag is used to group block-elements to format them with CSS.

The <span> tag is used to group inline-elements in a document. The <span> tag provides no visual change by itself. The <span> tag provides a way to add a hook to a part of a text or a part of a document.

Document structure

The head

The <head> element is a container for all the head elements. It can include a title for the document, scripts, styles, meta information, and more. The following elements can go inside the <head> element:

  • <title> (this element is required in an HTML document)
  • <style>
  • <base>
  • <link>
  • <meta>
  • <script>
  • <noscript>

Meta

Metadata is data (information) about data. The <meta> tag provides metadata about the HTML document. Metadata will not be displayed on the page, but will be machine parsable. Meta elements are typically used to specify page description, keywords, author of the document, last modified, and other metadata. The metadata can be used by browsers (how to display content or reload page), search engines, or other web services. HTML5 introduced a method to let web designers take control over the viewport (the user's visible area of a web page), through the <meta> tag.

  • <meta charset="UTF-8">
  • <meta name="description" content="Short description of this page">
  • <meta name="author" content="Lasse Hægland">
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">

Title

The <title> tag is required in all HTML documents and it defines the title of the document. The <title> element:

  • defines a title in the browser toolbar
  • provides a title for the page when it is added to favorites
  • displays a title for the page in search-engine results

The body

The <body> tag defines the document's body. The <body> element contains all the contents of an HTML document, such as text, hyperlinks, images, tables, lists, etc.

            </aside>

        </footer>  
    </body>
</html>
